`Forked Road` 是一个《杀戮尖塔 2》的联机玩法模组。
它的目标是在多人地图投票出现分歧时，不再强行把所有玩家压到同一条战斗路线，而是把不同投票结果拆分为独立分支来结算。

模组会记录每位玩家当前所属的分支，只让真正进入该分支的玩家参与对应房间与战斗逻辑，并在整个分支批次全部完成后，再把所有人带回共享地图流程继续推进。

## 核心功能

- 分支化路线投票：当多人投票指向不同地图节点时，模组会按目标节点分组，生成一个分支批次，而不是直接覆盖少数票。
- 并行分支流程：每个分支都可以进入自己的房间上下文并独立推进，同时保留联机同步流程。
- 只读旁观系统：提前完成、当前不在本分支内、或已经出局的玩家，可以旁观其他仍在进行中的分支，但不会对其进行操作。
- 按分支人数结算战斗：战斗缩放和参与者逻辑只基于实际进入该分支的玩家，而不是整队人数。
- 全批次完成后回到共享地图：只有当本轮所有分支都处理完成后，队伍才会统一回到共享地图并继续下一轮投票，也可以在下一轮自然汇合。
- 支持分支存档与读档恢复：模组会保存分支运行时状态，并在多人读档后更安全地恢复分支流程。
- 扩展房间支持：当前分支流程和旁观支持已经覆盖战斗、事件类房间、宝箱房、商店和休息点。

## 当前玩法流程

1. 所有玩家先在共享地图中正常投票。
2. 如果投票出现分歧，模组会锁定本轮分支批次，并按目标节点分组。
3. 每个分支只让本分支玩家进入对应房间并推进自己的流程。
4. 提前完成的玩家会进入等待或旁观状态，而不是单独推进整局流程。
5. 当本轮所有分支都结算完成后，所有玩家再一起回到共享地图进行下一次投票。

## 重要说明

- 这是一个会直接影响联机流程的玩法模组，建议房间内所有玩家使用相同版本。
- 分支内出局的玩家现在不再依赖“强制并入另一个正在进行中的分支”来维持流程。
- 旁观视图设计为只读，用于查看其他分支的状态，而不是远程控制其他房间。
- 当前仓库包含两套源码：
  - `beta/**` 是当前最新游戏版本使用的主实现。
  - `src/**` 是旧版游戏移植分支，用于兼容和参考。

## 玩家安装说明

### Windows

1. 获取模组文件，并确认最终文件夹中至少包含 `ForkedRoad.dll` 和 `ForkedRoad.json`。
2. 将整个 `ForkedRoad` 文件夹复制到 `<Slay the Spire 2>/mods/` 目录下。
3. 启动游戏后，模组会自动加载。

## 联机兼容性说明

1. 同一个联机房间中的所有玩家应尽量使用同一个模组构建版本。
2. 如果混用不同版本，可能会导致地图投票、房间同步、分支推进、旁观状态或读档恢复出现异常。
3. 当前正常使用不需要额外配置文件。

## 仓库说明

- 当前打包元数据里的版本号是 `1.0.8`。
- `1.0.8` 版本包含了较大规模的重构，尤其集中在分支运行时、旁观视图和存档恢复相关部分。
- 如果你要为当前最新游戏版本从源码构建，请使用 `ForkedRoad.csproj`。
- 如果你要处理旧版游戏移植分支，请使用 `ForkedRoad.Legacy.csproj`。
