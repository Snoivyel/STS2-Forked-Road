`Forked Road` 是一款《杀戮尖塔2》的联机玩法模组，用来让多人地图投票在遇到不同战斗路线选择时，不再被强行压成同一条路线，而是按投票结果拆分成多个分支逐个结算。

模组会记录每位玩家当前所在的路线，并根据进入该分支的玩家数量处理房间逻辑、战斗缩放与后续同步。等本轮所有分支处理完毕后，队伍会重新回到共享地图流程继续前进。

## ✨ 核心功能

- 分叉路线结算：当联机玩家把票投向不同地图节点时，模组会按目标节点分组，依次进入各自的战斗分支，而不是直接覆盖少数票。
- 按分支人数缩放战斗：每个分支只按实际进入该分支的玩家结算多人战斗逻辑，避免把未参与该分支的玩家也算进去。
- 非当前分支玩家旁观或等待：未进入当前分支的玩家会进入等待或旁观流程，不会错误介入其他分支的房间结算。
- 商店、宝箱与共享事件支持：分支流程下会处理商店、宝箱以及部分共享事件中的同步与视角切换，尽量保持联机体验连贯。
- 支线团灭自动并线：如果某个支线分支全员倒下，模组会尝试把该分支玩家并入其他仍在进行的分支继续流程。
- 分支完成后恢复共享地图：当本轮所有分支都结算完成后，联机地图与投票状态会恢复到正常的共享模式。

## 📦 玩家安装说明

### Windows

1. 获取模组文件，并确保最终文件夹内至少包含 `ForkedRoad.dll`、`ForkedRoad.json`
2. 将整个 `ForkedRoad` 文件夹复制到游戏的 `<Slay the Spire 2>/mods/` 目录下。
3. 启动游戏后，模组会自动加载。

### 联机注意事项

1. 这是一个会直接影响联机流程的玩法模组，建议房间内所有玩家都安装相同版本。
2. 如果只有部分玩家安装，或者各自版本不一致，可能会导致地图投票、房间同步或分支结算异常。
3. 当前版本没有额外配置文件，安装后即可直接使用。

## 🧭 机制说明

- 该模组主要处理“多人地图投票出现不同目标节点”的情况。
- 进入分支后，当前分支玩家会正常进入房间；其余玩家会被限制旁观流程中，直到轮到自己的分支，或者本轮分支全部结算完成。
- 某些分支在结束时会触发重新汇合逻辑，模组会同步玩家位置、房间状态以及后续地图投票。
- 当前项目重点覆盖战斗路线分叉场景，属于直接影响联机玩法的内容，建议和固定联机队伍一起使用并保持版本一致。

## ⚠️ 已知问题

1. 暂时不支持 `SL`。执行 `SL` 后，所有玩家的分支状态会被重新汇总到当前场景对应的同一路线中。
2. 在合作事件中，`物理机互肘` 和 `历史书任务` ，如果在分路后进入战斗，当前版本可能出现无法正常结束回合的问题；目前只能通过 `SL` 勉强继续流程。
