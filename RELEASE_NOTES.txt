Release Notes for v1.0.8

English

This update is a major gameplay-flow and stability refactor for Forked Road.
The project now uses a clearer branch runtime model and a more robust split-batch flow for multiplayer route divergence.

Key changes:

- Rebuilt the split-route runtime around explicit batch, branch, player, and spectator state.
- Refactored multiplayer route resolution into a clearer flow:
  shared map vote -> batch lock -> branch-local room flow -> unified return to shared map.
- Added stronger read-only spectator support for active remote branches.
- Spectator views now better support combat, event-like rooms, treasure rooms, merchants, and rest sites.
- Added branch and player target switching for spectator mode.
- Improved split-run save/load handling, including restore-state buffering and safer restore-time validation.
- Improved handling for stale act-transition restore data so outdated branch state is stripped instead of corrupting the loaded run.
- Improved RunLocation and room-scoped synchronization behavior for the latest beta game APIs.
- Added or expanded spectator-state and shared-event synchronization messages.
- Improved branch completion flow, branch-entry persistence, and branch-local synchronization behavior.
- Updated packaged DLL/PDB outputs to match the current verified beta build.

Behavior notes:

- Players who finish a branch early now move into waiting or spectator flow instead of advancing the run alone.
- Eliminated players no longer depend on forced mid-run branch merging as the primary way to keep the run progressing.
- The repository now clearly separates the active latest-game implementation in beta/** from the legacy old-game port in src/**.

Chinese

本次更新是一次偏底层的玩法流程与稳定性重构，重点加强了 Forked Road 在多人分路场景下的运行时结构、旁观体验和读档恢复能力。

主要更新内容：

- 重新构建了分路运行时模型，明确拆分为批次、分支、玩家和旁观状态。
- 将多人分路流程整理为更清晰的结构：
  共享地图投票 -> 锁定分支批次 -> 分支内房间推进 -> 全部分支完成后统一回到共享地图。
- 加强了远端分支的只读旁观支持。
- 旁观视图现在更完整地覆盖战斗、事件类房间、宝箱房、商店和休息点。
- 新增或完善了旁观时按分支、按玩家切换视角的能力。
- 强化了分支存档/读档恢复流程，包括 restore 状态缓冲和更安全的恢复校验。
- 改进了跨 Act 过渡时的陈旧恢复数据处理，避免旧分支状态污染新读取的存档。
- 针对最新 beta 游戏接口，改进了 RunLocation 与房间作用域同步逻辑。
- 新增或扩展了旁观状态与共享事件相关的同步消息。
- 优化了分支完成、分支进入后的保存，以及分支本地同步流程。
- 更新了随包发布的 DLL/PDB，使其与当前验证通过的 beta 构建保持一致。

行为说明：

- 提前完成分支的玩家现在会进入等待或旁观流程，而不是单独推进整局。
- 分支内出局玩家不再依赖“运行中强制并线”作为主要推进方式。
- 仓库结构现在更明确地区分为：
  - `beta/**`：当前最新游戏版本实现
  - `src/**`：旧版游戏移植分支
